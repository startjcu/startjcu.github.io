文档流

Normal Flow，常规浮动，内联元素(行内元素)从左往右排，块级元素从上往下排

块级元素的高度若无具体指定，则由其内部的文档流决定，宽度默认为body宽度

内联元素的高度由行高决定，宽度由内容、padding、margin和border决定，无法自定义设置

文字的对齐方式为基线(四线三格中的第三条线)对齐，不同的字体有不同的设计行高，布局时要特别注意

脱离文档流

绝对定位、固定定位(position: absolute/fixed)和浮动(float: left)会脱离文档流

HTML中的空格

HTML会将内联元素之外的空格全部删去(trim)，内联元素之间的多个空格会合并为一个空格

若要添加多个空格，须用转义字符nbsp;(no break space)

居中

内联元素水平居中，在父元素上加文本居中

内联元素垂直居中，父元素上使用行高和上下填充

块级元素尽量不要直接指定高度，否则多行会出现溢出，导致排版错乱；一般由其内部文档流决定它的高度

margin合并

那么子元素的margin会与父元素的margin合并，且不会增加父元素的高度

如果父元素指定了高度，就优先选择在父元素上使用flex布局

AJAX

定义

异步的JavaScript和XML：Asynchronous Javascript And XML

使用

1、使用XMLHttpRequest发请求

2、服务器返回XML格式的字符串

3、JS解析XML，并更新局部页面

现在使用更轻量级的JSON代替XML

跨域资源共享：Cross-Origin Resource Sharing

AJAX请求是严格遵守同源策略的，若要突破同源策略，就需要用到CORS跨域

JS-JSONP

发表于 2019-02-23

跨域请求

form表单

form表单请求源码

缺点：必须刷新或新开一个页面

<img>发送请求

img发送请求源码

缺点：只能发送GET请求，服务器必须返回一个图片才能请求成功

<script>发送请求

script发送请求源码

缺点：这种请求方式要求前端程序员对后端接口很清楚才能正确定义请求路径

JSONP

JSONP底层实现源码

在script的请求(src)后面添加callback查询参数

JSONP实现原理

1、请求方(一个网站的前端-浏览器)创建script，src指向响应方，同时传一个查询参数?callbackName=xxx

2、响应方根据查询参数callbackName，构造形如xxx.call(undefined,'你要的数据')，即xxx(result)的数据返回给客户端

3、浏览器接收到响应，就会执行xxx.call(undefined, ‘你想要的数据’)

4、浏览器将执行结果渲染出来，展示给用户，请求方就获得了他想要的数据

业界约定：1、callbackName一般用callback代替，2、xxx一般为随机数

jQuery实现JSONP

jQuey底层会封装请求参数无需单独指定

其他相关

1、JSONP为什么不支持POST请求

因为JSONP是通过动态创建script实现跨域请求的，只能用GET，不能用POST

2、JSONP与JSON无关

一般服务器端执行callback函数会将参数以JSON的格式传入，加上两端Padding，故称JSONP

3、JSONP与AJAX无关

因为jQuery将JSONP的实现封装在ajax属性/函数中，但AJAX的出现是在JSONP之后的
