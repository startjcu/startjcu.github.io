BFC

定义

块级格式化上下文：Block Formatting Context

浮动、绝对定位元素、非块盒的块容器(display属性为inline-block、table-cell等)

overflow不为visible(默认值)的块盒会为它的子元素建立一个新的块级格式化上下文

可以用来解决浮动子元素造成父级坍塌和margin(父元素不包含子元素上边距)合并的问题，但不推荐

同一个块级格式化上下文中的相邻块级盒（浮动元素不是块级盒）之间的竖直margin会合并，以较大的为准

触发BFC推荐方式

BFC的作用

1、用BFC包住浮动元素(推荐使用clearfix)，2、用float+div做左定宽右自适应布局(可用Flex替代)

IFC

font-size

字号，可以理解为活字印刷中用来盛放字符的金属方块(容器)的尺寸

每种字体都有不同的设计师推荐行高，行内元素默认基线(baseline)对齐

提升线ascender > 大写高度capital-height > x高度x-height > 基线baseline > 下降线descender

line-height

行高(盒)，用来决定内联元素真实占据高度

line-height的值不带单位指的是相对于字号的倍数，带单位指具体的值

行盒内部元素垂直方向默认居中对齐，不同字体基线不同，可能会造成设定好的line-height有细微的变化

vertical-align

详细教程

一个很少用到的属性，取值为top指的是实际占据高度顶线对齐，middle并不是居中对齐

唯一用途：用来处理图片下面的空隙，因为图片也是行内元素，默认会与父元素的文字基线对齐，从而产生空隙

为什么不用inline-block布局

因为行内元素垂直方向上不容易对齐，且会因html书写的格式产生间隙，故用flex布局或float布局来替代

CSS-Grid布局

发表于 2019-03-13

网格布局

概念

CSS网格布局（Grid）是一个二维的基于网格的布局系统

相对于Flex主要是一维系统而言，Grid可以同时处理行和列

使用步骤

1、使用display: grid;将父元素定义为一个网格容器(grid container)

2、使用grid-template-columns和grid-template-rows来设置列和行的大小(和数量)

3、使用grid-template-areas和grid-area来定义和将子元素放入网格

相对单位

fr：设置列或行占剩余空间的一个比例(fraction分数)

auto：设置列宽或行高自动等于它的内容的宽度或高度

%：将列宽或行高设置为它的父容器宽度或高度的百分比

grid container的属性

定义网格容器

设置模板行高和列宽

用repeat方法减少重复，如创建一个五列网格

设置行高和列宽尺寸范围

个数可以使用auto-fill(不足尺寸自动填充至最小尺寸)，auto-fit(不足自动适配填充满一行)

设置不同网格间的行间距和列间距(gap)

grid-gap = grid-row-gap + grid-column-gap

设置所有子元素水平和垂直对齐方式

给网格区域命名(.表示空单元格)

grid item的属性

使用网格线设置跨列和跨行(列的左边界线编号为1，行的上边界线编号为1，依次编号)

水平和垂直对齐方式(取值：start、center、end)

将子元素放置到网格区域中

响应式和Flex布局

发表于 2019-03-09

媒体查询

若只用max-width，要注意书写顺序，像素大的往后放

国内很少用响应式，一般用两个域名，或是用两套html和css

苹果手机默认用375像素的宽度去模拟980像素，然后进行缩放，故需要设置viewport

手机端交互方式的特点

1、没有hover，2、有touch事件，3、没有resize，4、没有滚动条

Flex布局

Flex之前的布局

1、文档流（正常流 Normal Flow）； 2、float + clear；

3、position：relative + absolute； 4、display：inline-block； 5、负margin

Flex布局特点

1、块级元素侧重垂直方向、行内布局侧重水平方向，flex布局与方向无关

2、flex布局可以实现空间自动分配、元素自动对齐（flexible：弹性、灵活）

3、flex适用于简单的线性布局，更复杂的布局要用到grid布局

Flex基本概念

主轴(main axis)：主尺寸(main size)、主轴起点(main start)、主轴终点(main end)

侧轴(cross axis)：侧尺寸(cross size)、侧轴起点(cross start)、侧轴终点(cross end)

子元素(flex item)、父元素(flex container需添加display: flex样式)

flex container的属性

1、方向：flex-direction

属性值：row默认值、column、row-reverse、column-reverse

2、换行：flex-wrap

属性值：nowrap默认值、wrap、wrap-reverse

3、流动：flex-flow

是方向和换行的复合属性：flex-flow = flex-direction + flex-wrap

4、主轴方向对齐方式：justify-content

属性值：flex-start默认值、space-between、center、space-around、flex-end

5、侧轴方向对齐方式：align-items

属性值：stretch默认值(将所有元素拉伸至最高元素的高度)、flex-start、center、flex-end

6、行与行之间隔多远：align-content

(很少用)属性值：stretch默认值、flex-start、flex-end、space-around、space-between

flex item的属性

1、增长比例：flex-grow

空间过多时，多余空间分配比例

2、收缩比例：flex-shrink

空间不够时，所缺空间分摊比例，一般不用

3、默认大小：flex-basis

一般不用，原始大小由内容大小决定，无需特意指定

4、复合属性：flex

flex = flex-grow + flex-shrink + flex-basis

5、属性：order

改变子元素排列顺序，默认值为0

6、自身的侧轴对齐方式：align-self

用于改变单独某个子元素的侧轴对齐方式，属性值同父元素的align-items属性

使用Flex布局

好玩的Flex小游戏

帮助青蛙跳到荷叶上

手机页面布局 (topbar + main + tabs)

上中下布局，上下高度固定(不用fixed定位)，中间自适应，可滚动实现要点

CSS宽度和高度

发表于 2019-03-02

文档流

Normal Flow，常规浮动，内联元素(行内元素)从左往右排，块级元素从上往下排

块级元素的高度若无具体指定，则由其内部的文档流决定，宽度默认为body宽度

内联元素的高度由行高决定，宽度由内容、padding、margin和border决定，无法自定义设置

文字的对齐方式为基线(四线三格中的第三条线)对齐，不同的字体有不同的设计行高，布局时要特别注意

脱离文档流

绝对定位、固定定位(position: absolute/fixed)和浮动(float: left)会脱离文档流

HTML中的空格

HTML会将内联元素之外的空格全部删去(trim)，内联元素之间的多个空格会合并为一个空格

若要添加多个空格，须用转义字符nbsp;(no break space)

长单词遇到阻碍自动换行

单行文字两端对齐

清除浮动

多行文本转一行带...

指定保留若干行，末尾带...

居中

内联元素水平居中，在父元素上加文本居中

内联元素垂直居中，父元素上使用行高和上下填充

块级元素尽量不要直接指定高度，否则多行会出现溢出，导致排版错乱；一般由其内部文档流决定它的高度

margin合并

父div与子div间若无阻隔(border、padding或内联元素等)，

那么子元素的margin会与父元素的margin合并，且不会增加父元素的高度

outline/border

outline不会改变盒模型的大小，等价于对border加了以下属性

块级元素水平居中

块级元素垂直居中一般在父元素上加上下填充

如果父元素指定了高度，就优先选择在父元素上使用flex布局
