JS类型转换与内存

类型转换

转字符串

1、toString()方法
除null和undefined以外的数据类型都有此方法
对象使用此方法获得的是"[object Object]"

2、String(data)方法
可以将任意类型的数据转换成字符串

3、在数据后面加空字符串+''
因为+只操作相同类型的数据，故其他类型与字符串相加时
会先调用String()方法将其他类型的数据转换成字符串再运算

转布尔值

1、Boolean(data)方法

2、在数据前加两个逻辑非运算符!!

五个falsy值：0、NaN、''、null、undefined

转数字

1、Number(data)方法
这种方法将字符串转成数字只能转纯数字字符串，否则会得到NaN(Not a Number)

2、parseInt(data)、parseFloat(data)方法
解析数字字符，按从左到右的顺序能解析多少个就解析多少个，遇到不能解析的就返回已解析的结果

3、减零法-0
底层使用的转换方法是Number()

4、取正法
在数据前加+号，即可将其转换成数字，底层使用的也是Number()方法

内存

内存和硬盘

内存，断电即失，读写数据快

硬盘，持久存储，读写速度慢，如SSD固态硬盘、机械硬盘

JS内存

浏览器会将电脑分配给JS的内存氛围代码区和数据区

数据区又划分为：
1、栈内存(Stack)
用来存储基本数据类型和64位内存地址(对象的引用)

2、堆内存(Heap)
因为对象可扩展属性，存在栈内存中扩展很消耗性能(类似向数组中插入数据)

故将引用数据类型存储在堆内存中，通过引用操纵对象
‘’’js
var a = { n: 1 } //a = ADDR45
var b = a //b = ADDR45
/**浏览器执行顺序从左到右

  • 但赋值运算符运算顺序从右到左
  • 故a.x = ADDR45.x, a = { n: 2 }-ADDR55, a.x = ADDR55 */
    a.x = a = { n: 2 }
    console.log(a.x) //ADDR55.x = undefined
    console.log(b.x) //ADDR45.x = { n: 2 }
    ‘’’
    垃圾回收(GC)
    如果一个对象没有被引用，他就是垃圾，将被回收
    ‘’’js
    var fn = function () { } //函数1

document.body.onclick = fn

fn = null//此时函数1还不是垃圾，因为还有document.body.oclick引用它

document.body.onclick = null//此时函数1已不被引用，就会视作垃圾进行回收

浅拷贝，拷贝的是引用
