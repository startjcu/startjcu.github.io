HTTP基础知识

页面相关术语

URL：Uniform Resource Locator

统一资源定位符，用来定位页面

HTTP：HyperText Transfer Protocol

超文本传输协议，用来下载页面

HTML：HyperText Markup Language

超文本标记语言，用来显示页面

资源相关术语

URI：Uniform Resource Identifier

统一资源标识符，有两种实现方式URL和URN

URL：Uniform Resource Locator

统一资源定位符，用来定位一个资源，一般指代网址，但文件移动后就找不到了

URN：Uniform Resource Name

统一资源名称，用来唯一标识某个资源，如ISBN国际标准书号International Standard Book Number

URL的组成

协议：(Protocal)https://

域名：(Host)startjcu.com，其中com为顶级域名、startjcu为二级域名

端口：(Port):443，对应服务器上的某个Web服务，http-80为默认端口

路径：(Path)/resum

参数：(Parameter)?id=1&name=star

锚点：(Hash)#3，用来实现某个页面内不同位置的跳转

DNS服务

域名系统：Domain Name System，用来将域名解析为对应的IP地址

流程：域名 > (本地hosts)电信DNS服务 > IP地址

服务器(Server电脑) > HTTP > 客户端(Client浏览器)

HTTP的作用是置动服务器和浏览器之间如何通信

HTTP请求

curl -X POST -s -v -- 'https://www.baidu.com'

curl：command line url，此命令用于在命令行实现url请求

-X：request method，指定请求方法

-s：silence，不显示请求进度条

-v：verbose，详细地显示请求>和响应<

HTTP方法

GET：获取数据

POST：上传(提交)数据

PUT/PATCH：更新(整体/局部)数据

DELETE：删除数据

其他方法：HEAD、OPTIONS等

请求报文格式

1、请求方法 路径及参数 协议/版本号

2、key : value键值对，如Host: www.baidu.com

3、空行，用来分隔2和4

4、要上传的数据，如用户名、密码等

响应报文格式

1、协议/版本号 状态码 状态说明

2、key : value键值对，如Content-Type: 141

3、空行，用来分隔2和4

4、下载的数据

状态码
1xx：消息，如100-Continue，表示还需要进一步的操作

2xx：成功，如200-OK，表示普通成功；201-Created，表示创建成功

3xx：重定向，如301-Moved Permanently；302-Move Temporarily

4xx：请求错误，如401-Unauthorized未登录；404-Not Found资源丢失

连接相关术语

TCP：Transmission Control Protocol

传输控制协议，每次建立连接前，客户端和服务器都要进行三次成功对话后，才正式开始传输内容(TCP三次握手)

UDP：User Datagram Protocol

用户数据报协议，和TCP的区别是，TCP可靠，面向连接，相对UDP较慢，而UDP正好相反

IP：Internet Protocol

网络协议，每个电脑都应该有一个独立的IP地址。127.0.0.1/localhost指本机，0.0.0.0不指向任何设备

外网IP > 路由器 > 内网IP

路由器(必由之路)，用于给信息指路/进行内容分发，故又称网关

HTTP是基于TCP/IP协议构建的，使用HTTP协议访问另一个主机时，必须提供IP地址和端口号

小知识：argc=arguments count、argv = arguments value
